1. Giới thiệu

1.1. Mục đích

Hệ thống nhắc nhở nhân viên (assignee) báo cáo tiến độ các task đang thực hiện thông qua Web Push Notification (FCM), đồng thời cung cấp trang nhập tiến độ hàng loạt giúp user xử lý nhanh nhiều task cùng lúc trong ngày.

  • Nhắc tự động qua trình duyệt (kể cả khi không mở tab) tại 4 mốc cố định: 08:30, 11:30, 14:30, 17:30
  • Áp dụng đồng nhất cho toàn hệ thống — không cấu hình per company
  • Chỉ nhắc task IN_PROGRESS có rule báo cáo active, đến lịch hôm nay và chưa nhập tiến độ

1.2. Phạm vi

Trong scope: Đăng ký FCM token, cron phát hiện task chưa báo cáo và gửi push, trang nhập tiến độ hàng loạt, badge count sidebar, click notification navigate.

Ngoài scope: Push mobile (FCM topic riêng), email, cấu hình giờ nhắc per company/user, thay đổi cấu trúc task_progress_report_rules.

5. Use Cases chi tiết

UC-01: Đăng ký nhận push notification (FCM Token)

ActorNhân viên
PreconditionUser đã login; trình duyệt hỗ trợ Notification + Service Worker (HTTPS bắt buộc trừ localhost)
Main Flow1. User mở app sau login
2. FE đăng ký SW firebase-messaging-sw.js
3. FE gọi Notification.requestPermission() → user "Cho phép"
4. FE gọi getToken() → fcm_token
5. FE gửi POST /assign/fcm-tokens với {token, device_info}
6. BE updateOrCreate token vào fcm_tokens (key: token)
7. BE trả 200 success
PostconditionToken gắn với employee_id → cron có thể gửi push
AlternativeUser từ chối permission → không gửi token, không nhận push (FE silent skip)
ExceptionToken đã tồn tại → upsert update employee_id mới

UC-02: Hệ thống tự động gửi push notification (Cron)

ActorHệ thống (Cron)
Preconditionschedule:run được system cron gọi mỗi phút; FCM credentials đã cấu hình FIREBASE_CREDENTIALS
Main Flow1. Schedule trigger assign:notify-task-report mỗi 30 phút
2. Command check now ∈ {08:30, 11:30, 14:30, 17:30}? Không → return 0
3. Query Task IN_PROGRESS + rule active; eager load progressLogs(today, pct>0)
4. Filter due today theo cycle_type (daily/weekly/monthly)
5. Skip task có progressLogs.isNotEmpty()
6. Group theo assignee_id → đếm số task
7. Gửi FCM message: notification + data + webpush (fcm_options.link)
8. NotFound → xoá token
9. Log "Sent: X, Failed: Y"
PostconditionUser chưa nhập tiến độ nhận push notification
AlternativeKhông có user cần nhắc → log "Không có user nào"
ExceptionFirebase không khả dụng → log error, return 1; withoutOverlapping() đảm bảo không double-send

UC-03: User nhận và click vào notification

ActorNhân viên
PreconditionUC-01 thành công, UC-02 đã gửi push
Main Flow1. FCM đẩy message tới browser
2. Foreground: onMessage → toast trong app
3. Background: SW nhận → browser hiện system notification
4. User click notification
5. SW custom notificationclick handler chạy trước Firebase + stopImmediatePropagation()
6. SW clients.openWindow(fcm_options.link) hoặc focus tab existing → navigate /assign/tasks/daily-report
PostconditionUser được đưa đến trang nhập báo cáo
ExceptionToken expired → BE auto-delete ở UC-02 step 8, lần sau không gửi

UC-04: Xem danh sách task cần báo cáo hôm nay

ActorNhân viên
PreconditionUser login, truy cập /assign/tasks/daily-report
Main Flow1. FE gọi GET /assign/tasks/daily-report
2. BE lấy task của auth()->id() có IN_PROGRESS + rule active
3. Filter due today
4. Generate progress_rows: past + today + future, gắn log đã có
5. Set flag is_reported_today nếu hôm nay đã có log pct>0
6. Sort: chưa nhập trước, theo send_time tăng dần
7. FE render summary + accordion collapse + bảng progress_rows
AlternativeKhông có task → empty state "Không có task cần báo cáo tiến độ hôm nay"

UC-05: Nhập và lưu báo cáo tiến độ hàng loạt

ActorNhân viên
PreconditionUC-04 đã load; task IN_PROGRESS của bản thân
Main Flow1. Expand task → hiện progress_rows
2. Chỉ dòng today editable; past readonly; future disabled
3. Nhập hours, progress_pct, note ≥ 1 task
4. Click "Lưu tất cả"
5. FE validate: sum past + today ≤ 100% (vượt → toast error + highlight)
6. Build payload chỉ task có dữ liệu today
7. POST /daily-report/save
8. BE validate lại: ownership + report_date=today + ≤ 100%
9. updateOrCreate task_result_progress_logs (key: task_id + report_date)
10. Sync task.progress_pct = latestLog.progress_pct
11. Trả {success, errors}
12. FE toast success + reload
PostconditionTiến độ lưu; user không bị nhắc lần kế (pct>0)
ExceptionVượt 100% → BE skip task đó, các task khác vẫn lưu

UC-06: Xem badge count trên sidebar

ActorNhân viên
Main Flow1. FE gọi GET /daily-report/count (on-mount/định kỳ)
2. BE đếm task IN_PROGRESS của user, due today, pct=0
3. Trả {count: N}
4. FE hiện badge số N; N=0 ẩn
