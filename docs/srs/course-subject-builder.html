1. Giới thiệu & phạm vi

1.1. Mục đích

Rebuild màn tạo/sửa Khoá học (subjects) từ form đơn giản lên giao diện builder 4 tab đầy đủ. Đổi label "Môn học" → "Khoá học" trên toàn bộ màn subjects. Chuẩn hoá dữ liệu backend bằng các bảng mới thay cho JSON evaluation_config cũ. Bổ sung ảnh banner, 3 field mô tả chi tiết, cấu hình chứng chỉ và cấu hình người học polymorphic.

1.2. Trong phạm vi

  • Màn danh sách khoá học: filter, phân trang, khoá/mở khoá, xoá.
  • Màn tạo/sửa — builder 4 tab: Thông tin (banner, chương/bài, 3 field mô tả) · Đánh giá (completion rule hoặc exam) · Người học (onboarding, bắt buộc/khuyến nghị) · Chứng chỉ (upload template, canvas preview, PDF).
  • Lưu tạm (DRAFT) — chỉ cần name, bỏ qua validation chi tiết.
  • Upload ảnh banner và ảnh nền chứng chỉ qua endpoint chung POST /files/upload (AWS S3).
  • Migration dữ liệu cũ: evaluation_config JSON → subject_exams; working_position_subjects + capability_subjects → subject_assignees.

1.3. Ngoài phạm vi

  • Export Excel (đã có, không thay đổi).
  • Job onboarding auto-assign chạy nền.
  • Render PDF chứng chỉ chính thức (chỉ preview + download cục bộ).
  • Màn downstream: training_programs, courses, training_plans, reports — giữ nguyên logic.
  • Permission phân cấp theo phòng ban/chi nhánh (chỉ dùng permission toàn công ty).

5. Danh sách Use Case chi tiết

UC-01 — Xem danh sách khoá học
Actor
Admin / Manager Đào tạo
Precondition
Đã đăng nhập, có quyền Quản lý khoá học.
Main Flow
1. Truy cập /training/subjects.
2. Hệ thống gọi GET /api/v1/training/subjects với filter: tên/mã, loại đào tạo, trạng thái (1/2/3), phân trang.
3. Bảng hiển thị: mã, tên, loại đào tạo, trạng thái badge (xanh/đỏ/xám), nút sửa, khoá/mở khoá, xoá.
Postcondition
Danh sách hiển thị đúng filter.
Alt / Exception
Không có dữ liệu → empty state. Token hết hạn → redirect login.
UC-02 — Tạo mới khoá học
Actor
Admin / Manager Đào tạo
Precondition
Có quyền Quản lý khoá học.
Main Flow
1. Click "+ Tạo mới" → chuyển sang /training/subjects/add.
2. Builder mở với 4 tab: Thông tin Đánh giá Người học Chứng chỉ.
3. Điền tối thiểu name, cấu hình các tab theo nhu cầu.
4. Click "Lưu" → validate đầy đủ → POST /subjects/store-builder với status=1.
5. BE tạo subject + auto-gen SUB-YYYY-XXXX nếu code rỗng + sync tất cả relation trong 1 transaction.
6. Redirect về danh sách, toast thành công.
Postcondition
Subject mới tạo trong DB, status=1. subject_exams, subject_assignees, subject_certificate_fields được insert đúng.
Alt / Exception
Validation lỗi → FE hiển thị lỗi inline, tự chuyển đúng tab có lỗi. Mã trùng → 422 Mã khoá học đã tồn tại.
UC-03 — Chỉnh sửa khoá học
Actor
Admin / Manager Đào tạo
Precondition
Khoá học tồn tại, status = Hoạt động (1) hoặc Nháp (3). Khoá (2) → nút sửa disabled.
Main Flow
1. Click icon sửa → chuyển sang /_id/index.
2. FE gọi GET /subjects/{id}/builder → load toàn bộ dữ liệu 4 tab (chapters, exams, assignees, cert fields).
3. Chỉnh sửa, click "Lưu" → POST /subjects/{id}/builder.
4. BE update subject + re-sync tất cả relation trong transaction; cập nhật updated_by, updated_at.
Postcondition
Subject được cập nhật. Tất cả relation được sync lại (wipe + re-insert).
Alt / Exception
Subject không tồn tại → 404.
UC-04 — Lưu tạm (DRAFT)
Actor
Admin / Manager Đào tạo
Precondition
Đang ở màn tạo hoặc sửa khoá học.
Main Flow
1. Click "Lưu tạm".
2. FE validate duy nhất: name không rỗng.
3. Gọi API với status=3. BE chỉ áp rules: name required, code unique.
4. Khoá học được lưu với status=3 (Nháp). Nút sửa vẫn hiển thị.
Postcondition
Subject status=3. Các bảng con được sync theo dữ liệu gửi lên (có thể rỗng).
Alt / Exception
name rỗng → FE báo lỗi inline dưới field, chuyển về Tab 1, không gọi API.
UC-05 — Cấu hình chương & bài học (Tab 1)
Actor
Admin / Manager Đào tạo
Precondition
Đang trong builder, Tab "Thông tin".
Main Flow
1. Toggle use_chapters=1 (Có chương): thêm chương → thêm bài từ ngân hàng vào chương → kéo thả sắp xếp.
1a. Toggle use_chapters=2 (Không có chương): bài học thẳng vào khoá học.
2. Với mỗi bài, click ⓘ → cấu hình override_completion, prerequisite.
3. Đổi use_chapters 1→2: BE xoá tất cả chapters, set chapter_id=null cho lessons.
4. Xoá bài học đang là prerequisite → BE clean prerequisite_subject_lesson_ids của bài còn lại.
Postcondition
subject_chapters và subject_lessons được sync đúng thứ tự sort_order.
Business Rule
Khi use_chapters=1, mỗi chương phải có ≥1 bài (validate khi Lưu status≠3).
UC-06 — Cấu hình đánh giá (Tab 2)
Actor
Admin / Manager Đào tạo
Main Flow — Completion
1. Chọn evaluation_mode=completion.
2. Chọn rule: all_required / percent / weighted.
3. Nếu rule=percent → nhập percent (1–100) + percent_mode.
Main Flow — Exam
1. Chọn evaluation_mode=exam.
2. Chọn ≥1 đề thi. Mỗi đề: nhập time_limit_min, pass_score_percent, chọn người chấm nếu đề có tự luận.
3. Nhập exam_attempt_limit, chọn exam_score_rule (highest/last/average).
4. Toggle exam_participation_required → nhập exam_min_required_percent (1–100).
Postcondition
subject_exams + subject_exam_graders được sync. Đổi mode → wipe dữ liệu mode cũ.
Alt / Exception
Exam mode, không có đề → 422. Đề có tự luận, không có grader → FE validate.
UC-07 — Cấu hình người học (Tab 3)
Actor
Admin / Manager Đào tạo
Main Flow
1. Toggle onboarding_enabled=1 → nhập onboarding_new_employee_days và onboarding_must_finish_days (0 = không giới hạn).
2. Khối "Bắt buộc": chọn loại (Phòng ban / Chức vụ / Năng lực) → multi-select → chip list. Set is_mandatory=1.
3. Khối "Khuyến nghị": tương tự, is_mandatory=0.
4. Click × chip để xoá đối tượng đã chọn.
Postcondition
subject_assignees được sync đúng assignee_type + assignee_id + is_mandatory.
Business Rule
onboarding_new_employee_days=0 = luôn coi mới; onboarding_must_finish_days=0 = không deadline.
UC-08 — Cấu hình chứng chỉ (Tab 4)
Actor
Admin / Manager Đào tạo
Main Flow
1. Toggle certificate_enabled=1.
2. Upload ảnh nền → URL lưu vào certificate_template_url.
3. Cấu hình 4 trường: course_name, employee_name, issued_date, signer (text, X, Y, font_size, font_weight).
4. Canvas preview 1600×900 render real-time. Button "Download PDF" → jsPDF.
Postcondition
certificate_enabled, certificate_template_url, 4 rows trong subject_certificate_fields được lưu.
Business Rule
certificate_enabled=1 → certificate_template_url required + certificate_fields phải đủ 4 trường (size:4).
UC-09 — Khoá / Mở khoá khoá học
Actor
Admin / Manager Đào tạo
Main Flow
1. Click icon khoá/mở khoá ở danh sách → BaseConfirmModal hiện với title/message động.
2. Xác nhận → GET /subjects/{id}/lock hoặc /unlock.
3. Danh sách refresh; badge trạng thái cập nhật (xanh ↔ đỏ).
Postcondition
status chuyển 1→2 (khoá) hoặc 2→1 (mở).
UC-10 — Xoá khoá học
Actor
Admin
Precondition
is_can_delete=true.
Main Flow
1. Click icon xoá → confirm dialog → DELETE /subjects/{id}/builder.
2. Subject + các bảng con bị xoá.
Business Rule
DRAFT (status=3) luôn xoá được. Status≠DRAFT không xoá được nếu có reference trong: course_requests, training_requests, training_planning_subjects, training_program_subject, courses.

10.1. Component: BannerUploader

File: hrm-client/components/shared/BannerUploader.vue

  • Emit: change(File), remove().
  • Behavior: Chưa có ảnh → dashed border, icon + hint text, click toàn vùng mở picker. Có ảnh → full-width img, hover overlay "Đổi ảnh" + nút xoá đỏ, spinner khi uploading. FileReader preview local ngay khi chọn; sau khi server trả URL → switch sang URL chính thức.

10.2. Component: BulletListEditor

File: hrm-client/components/shared/BulletListEditor.vue

  • Engine: CKEditor 5 minimal (window.CKEDITOR.replace()) — toolbar: B / I / U / BulletedList / Undo / Redo. Không dùng $loadCKEditor plugin để tránh bị override toolbar.

11. Non-functional requirements

  • Performance: List API <500ms với dataset thông thường (<10.000 subjects). Builder GET load relation lazy.
  • Atomicity: storeWithStructure / updateWithStructure bọc DB::transaction — rollback toàn bộ nếu bất kỳ sync method nào fail.
  • Security: JWT (tymon/jwt-auth) + Laravel Sanctum. Permission Quản lý khoá học check tại middleware route group.
  • Concurrency: Không có race condition đặc biệt vì save overwrite toàn bộ relation (wipe + insert), không partial update.
  • Upload: Ảnh banner + cert template dùng endpoint chung /files/upload (AWS S3). CORS phải cấu hình để canvas crossOrigin='anonymous' hoạt động; có fallback render offscreen khi bị taint.
  • Compatibility: PHP 7.4 + Laravel 8.65 + MySQL; Node 14.21.3 + Nuxt 2.14 + Vue 2 + Bootstrap-Vue 2.15.
  • Browser: Chrome / Firefox / Edge / Safari (canvas API, FileReader API, drag-drop SortableJS).
  • PDF: jspdf package cần được install (npm i jspdf). Nếu chưa có → tính năng "Tải xuống PDF" fail (có toast lỗi, không crash app).
