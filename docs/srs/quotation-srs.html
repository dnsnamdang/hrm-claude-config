1. Giới thiệu

1.1. Mục đích

Quản lý toàn bộ vòng đời báo giá hàng hoá (Quotation) trong hệ thống ERP TPE: từ khi nhân viên kinh doanh tạo yêu cầu báo giá (YCBG) → nhân viên xây dựng giá lập báo giá dựa trên BOM List → duyệt đa cấp (tự duyệt / TP / BGĐ) → in báo giá cho khách hàng.

1.2. Phạm vi

Trong scope:

  • Tạo báo giá từ yêu cầu báo giá (1 YCBG → 1 Báo giá)
  • Xây dựng giá bán, giá nhập, VAT cho từng sản phẩm và dịch vụ bổ sung
  • Quy trình duyệt 3 cấp dựa trên giá trị đơn hàng & tỷ suất lợi nhuận
  • Xuất / Nhập Excel giá sản phẩm
  • In báo giá với tuỳ chọn cột hiển thị
  • Lịch sử thay đổi (audit trail)
  • Ghi chú kinh doanh (sales note) chỉnh sửa sau duyệt

Ngoài scope:

  • Quản lý BOM List (đã có SRS riêng)
  • Quản lý Yêu cầu báo giá / Pricing Request (module riêng)
  • Cấu hình ngưỡng duyệt giá (BomPriceApprovalConfig — màn cấu hình riêng)

3. Use Cases

3.0. Use Case Diagram

UC-01: Tạo báo giá
Actor
NV Xây dựng giá
Precondition
Có Yêu cầu báo giá (YCBG) ở trạng thái "Chờ xây dựng giá" (status=2)
Main Flow
1. Từ màn YCBG, bấm "Tạo báo giá"
2. Hệ thống lock YCBG (lockForUpdate) → tạo bản ghi quotations
3. Snapshot toàn bộ thông tin KH (tên, MST, địa chỉ, email, người LH, SĐT) từ YCBG
4. Copy danh sách sản phẩm từ BOM vào quotation_product_prices (estimated_price từ BOM)
5. Sinh mã tự động: BG-YYYY-NNNNN
6. Chuyển YCBG sang trạng thái "Đang xây dựng giá"
7. Redirect sang trang chỉnh sửa báo giá
Postcondition
Báo giá mới ở trạng thái Đang tạo (1), YCBG chuyển sang "Đang XD giá"
Exception
YCBG đã có báo giá → trả lỗi 422 (1 YCBG = 1 Báo giá)
UC-02: Sửa báo giá (xây dựng giá)
Actor
NV Xây dựng giá (người tạo)
Precondition
Báo giá ở trạng thái Đang tạo (1), user là người tạo
Main Flow
1. Mở trang sửa báo giá
2. Chỉnh sửa thông tin chung: hiệu lực, giao hàng, bảo hành, điều khoản thanh toán, ghi chú, tiền tệ
3. Nhập giá cho từng sản phẩm:
  - CHA: nhập quoted_price (giá bán) + vat_percent; estimated_price = auto roll-up từ CON (disabled)
    Công thức: CHA.estimated_price = SUM(con.estimated_price × con.qty) / cha.qty
  - CON: nhập estimated_price (giá nhập); quoted_price + VAT bị khoá, hiển thị "—"
  - ĐỘC LẬP: nhập cả 3 trường (estimated_price, quoted_price, vat_percent)
4. Thêm/sửa/xoá dịch vụ bổ sung (tự sinh mã DV-{BG code}-NNN)
5. Áp dụng VAT hàng loạt (tuỳ chọn: tất cả / chỉ hàng = 0%)
6. Dùng công cụ làm tròn giá (chục / đơn vị / 0.1)
7. Bấm "Lưu nháp" → lưu tất cả thay đổi
Postcondition
Giá sản phẩm, dịch vụ, thông tin chung được lưu. Tổng VAT + sau VAT được tính lại (cache)
Alternative
- Nhập giá từ Excel: UC-11
- Áp VAT hàng loạt: modal xác nhận, chọn mode "tất cả" hoặc "chỉ VAT=0"
- Lần đầu nhập VAT ≠ 0 → soft-prompt hỏi áp dụng cho tất cả hàng đang = 0%
UC-03: Xoá báo giá
Actor
NV Xây dựng giá (người tạo)
Precondition
Báo giá ở trạng thái Đang tạo (1), user là người tạo
Main Flow
1. Bấm "Xoá" → xác nhận
2. Cascade xoá: quotation_product_prices, quotation_service_items, quotation_histories
3. Nếu YCBG không còn báo giá nào → revert YCBG về "Chờ xây dựng giá"
Postcondition
Báo giá bị xoá khỏi hệ thống
UC-04: Xem danh sách báo giá
Actor
Tất cả (theo quyền view scope)
Main Flow
1. Vào menu "Quản lý báo giá"
2. Danh sách hiển thị với bộ lọc: công ty, phòng ban, bộ phận, NV tạo, dự án, giải pháp, version, khách hàng, trạng thái, cấp duyệt, người duyệt, khoảng ngày tạo
3. Tìm kiếm nhanh theo mã BG + tên KH
4. Tuỳ chỉnh cột hiển thị (lưu preference server)
5. Thao tác trên dòng: Xem | Sửa | In | Lịch sử | Xoá
Filter logic
- Quyền "Xem tất cả" → không lọc theo tổ chức
- Quyền "Theo công ty" → company_id = user.company_id
- Quyền "Theo phòng ban" → department_id = user.department_id
- Quyền "Theo bộ phận" → part_id = user.part_id
UC-05: Xem chi tiết báo giá
Actor
Tất cả (theo quyền)
Main Flow
1. Bấm xem 1 báo giá → trang chi tiết
2. Hiển thị: thông tin chung, bảng sản phẩm (nhóm/cha-con), dịch vụ bổ sung, tổng tiền, ghi chú
3. Cột giá nhập chỉ hiện nếu user có quyền can_view_import_price
4. Footer bar: nút Sửa (nếu status=1 & creator), Duyệt TP/BGĐ, Từ chối, Xuất Excel, In
UC-06: Trình duyệt báo giá
Actor
NV Xây dựng giá (người tạo)
Precondition
Báo giá ở Đang tạo (1), tất cả sản phẩm có giá > 0
Main Flow
1. Bấm "Trình duyệt" → mở modal xác nhận
2. Hệ thống tính toán cấp duyệt (1/2/3) dựa trên giá trị đơn hàng + tỷ suất LN
3. Hiển thị preview: tổng nhập, tổng bán, tỷ suất, cấp duyệt, workflow mô tả
4. User xác nhận → submit
Flow theo cấp
Cấp 1: "Bạn có thể tự duyệt" → bấm "Tự duyệt" → trạng thái = Đã duyệt (4)
Cấp 2: → Chờ TP duyệt (2) → TP duyệt → Đã duyệt (4)
Cấp 3: → Chờ TP duyệt (2) → TP chuyển tiếp → Chờ BGĐ duyệt (3) → BGĐ duyệt → Đã duyệt (4)
Postcondition
Trạng thái thay đổi theo cấp. Notification gửi tới người duyệt tương ứng
UC-07: Duyệt báo giá (TP)
Actor
Trưởng phòng
Precondition
Báo giá ở Chờ TP duyệt (2), user có quyền TP duyệt
Main Flow
1. Vào "Báo giá chờ duyệt" hoặc xem chi tiết
2. Review thông tin báo giá
3. Bấm "Duyệt"
4a. Cấp 2 → trạng thái = Đã duyệt (4)
4b. Cấp 3 → trạng thái = Chờ BGĐ duyệt (3), gửi notification BGĐ
Alternative
TP từ chối → UC-09
UC-08: Duyệt báo giá (BGĐ)
Actor
Ban giám đốc
Precondition
Báo giá ở Chờ BGĐ duyệt (3), user có quyền BGĐ duyệt
Main Flow
1. Xem chi tiết báo giá
2. Bấm "Duyệt" → trạng thái = Đã duyệt (4)
3. Cập nhật YCBG → "Đã có báo giá". Cascade cập nhật dự án + giải pháp
Alternative
BGĐ từ chối → UC-09
UC-09: Từ chối báo giá
Actor
TP hoặc BGĐ
Precondition
Báo giá ở Chờ TP (2) hoặc Chờ BGĐ (3)
Main Flow
1. Bấm "Từ chối" → mở modal nhập lý do (bắt buộc, max 1000 ký tự)
2. Hệ thống: reset trạng thái về Đang tạo (1), xoá thông tin duyệt
3. Gửi notification cho người tạo + TP (nếu BGĐ từ chối)
Postcondition
Báo giá quay về nháp, người tạo có thể sửa và trình duyệt lại
UC-10: Xuất Excel báo giá
Actor
NV XDG, NV KD
Main Flow
1. Trang chi tiết hoặc trang sửa → bấm "Xuất Excel"
2. Hệ thống tạo file Excel với: nhóm sản phẩm, cha-con, giá nhập, giá bán, VAT, tổng
3. Bao gồm dịch vụ bổ sung ở cuối
4. Download file .xlsx
UC-11: Nhập giá từ Excel
Actor
NV Xây dựng giá
Precondition
Báo giá ở Đang tạo (1)
Main Flow
1. Trang sửa → bấm "Nhập giá" → chọn file Excel
2. Hệ thống validate: match theo mã sản phẩm (code)
3. Cập nhật giá theo row type:
  - CON: chỉ estimated_price
  - CHA: quoted_price + vat_percent
  - ĐỘC LẬP: cả 3 trường
  - Mã bắt đầu "DV-": tạo/cập nhật dịch vụ bổ sung
4. Hiển thị kết quả: matched / updated / unmatched
UC-12: In báo giá
Actor
NV XDG, NV KD
Main Flow
1. Trang chi tiết hoặc trang danh sách → bấm "In"
2. Modal chọn cột: 14 cột tuỳ chọn + checkbox "Hiện hàng hoá cấp con"
3. Bấm "Xem trước" → modal preview full-screen
4. Layout in: banner công ty → tiêu đề "BÁO GIÁ HÀNG HÓA" → thông tin KH + BG → bảng sản phẩm (nhóm/cha-con/dịch vụ) → tổng → điều khoản → chữ ký
5. Bấm "In" → mở print dialog trình duyệt (A4 landscape)
Kỹ thuật
Hoàn toàn FE — dùng window.open() tạo cửa sổ mới với HTML + CSS inline → window.print().
Không cần API mới, dữ liệu đã load sẵn trên trang.

9.3. Công thức tính toán

9.3.1. Đơn giá nhập CHA (auto roll-up)

CHA.estimated_price = SUM(con.estimated_price × con.qty_needed) / CHA.qty_needed
Đơn giá nhập của sản phẩm CHA được tính tự động từ tổng thành tiền nhập các CON chia cho số lượng CHA. Trường này bị disabled trên UI, auto-recalculate mỗi khi CON thay đổi giá.

9.3.2. Công thức per-row

Chỉ sốCông thứcÁp dụng cho
Thành tiền nhập (lineImportTotal)estimated_price × qtyMọi loại: CHA, CON, ĐỘC LẬP, DV
Thành tiền bán (lineSaleTotal)quoted_price × qtyCHA + ĐỘC LẬP + DV. CON không có (hiện "—")
Tiền VAT (lineVatAmount)lineSaleTotal × vat_percent / 100CHA + ĐỘC LẬP + DV. CON = 0
Sau VAT (lineAfterVat)lineSaleTotal + lineVatAmountCHA + ĐỘC LẬP + DV
Tỷ suất LN per-row(lineSaleTotal − lineImportTotal) / lineImportTotal × 100CHA + ĐỘC LẬP + DV. Mẫu số = thành tiền nhập

9.3.3. Công thức tổng

Chỉ sốCông thứcGhi chú
Tổng nhập (totalImport)SUM(CON + ĐỘC LẬP: estimated_price × qty) + SUM(DV: estimated_price × qty)Bỏ qua CHA (vì CHA.estimated_price = roll-up từ CON, cộng cả 2 sẽ double-count)
Tổng bán (totalSale)SUM(CHA + ĐỘC LẬP: quoted_price × qty) + SUM(DV: quoted_price × qty)Chỉ parent-level + DV, loại trừ CON
Tổng VAT (totalVat)SUM(CHA + ĐỘC LẬP: lineVatAmount) + SUM(DV: lineVatAmount)Chỉ parent-level + DV
Tổng sau VAT (totalAfterVat)totalSale + totalVatCache vào quotations.total_after_vat
Tỷ suất LN% tổng(totalSale − totalImport) / totalImport × 100Mẫu số = tổng nhập. Tính trước VAT
Giá trị đơn hàng VNDtotalSale × exchange_rateQuy đổi VND cho tính cấp duyệt
Cấp duyệtMAX(findLevel(order_value, V_vnd), findLevel(profit_margin, M%))V = giá trị đơn hàng VND, M = tỷ suất LN%
