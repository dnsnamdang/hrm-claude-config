1. Giới thiệu & phạm vi

1.1. Mục đích

Tài liệu đặc tả luồng end-to-end từ lúc Nhân viên làm giá (NLG) tạo BOM List (Bill of Materials — danh mục hàng hoá/vật tư phục vụ định giá), qua bước Nhân viên Kinh doanh (NV KD) gửi Yêu cầu báo giá (YCBG), đến lúc Báo giá được duyệt bởi Trưởng phòng (TP) và/hoặc Ban Giám đốc (BGĐ) — sẵn sàng để NV KD bổ sung ghi chú và gửi khách hàng.

1.2. Trong phạm vi

  • Quản lý BOM List: tạo / sửa / xoá / import Excel / export Excel / phân quyền theo cấp tổ chức.
  • Yêu cầu báo giá (YCBG): tạo từ dự án tiền khả thi (TKT), auto-pick BOM tổng hợp đã duyệt, gửi NLG.
  • Báo giá: tạo từ YCBG (snapshot khách hàng + thương mại), nhập giá, quản lý VAT per-row + bulk apply, tính cấp duyệt tự động, workflow duyệt 3 cấp.
  • Thông báo in-app cho các bước chuyển trạng thái quan trọng.

1.3. Ngoài phạm vi

  • Gửi báo giá ra khách hàng qua email/PDF ngoài hệ thống.
  • Tích hợp thanh toán, hợp đồng điện tử.
  • Chỉnh sửa giá sau khi báo giá đã Đã duyệt (chỉ sửa sales_note).

4. Sơ đồ Use Case

Sơ đồ mô tả 5 actor tương tác với 12 use case chính của luồng BOM List → Duyệt Báo giá. Hình stick = actor; oval = use case; đường nối = mối liên quan.

5. Danh sách Use Case chi tiết

UC-01 — Tạo BOM List
Actor
NLG (hoặc user có quyền "Xây dựng giá Bom giải pháp")
Precondition
User đã login, có hạng mục/giải pháp cần BOM.
Main Flow
1. Mở /assign/bom-list/add → nhập: tên, loại (Thành phần / Tổng hợp), gắn hạng mục hoặc giải pháp, customer.
2. Thêm sản phẩm vào BOM (hỗ trợ cấu trúc cha–con).
3. Lưu → hệ thống sinh mã BOM-YYYY-NNNNN, status = 1 (Đang tạo).
4. NLG tiếp tục sửa, khi xong → submit duyệt → status chuyển tuần tự 2 (Hoàn thành) → 3 (Chờ duyệt) → 4 (Đã duyệt).
Postcondition
BOM có status 4 (Đã duyệt) mới được dùng cho YCBG.
Alt / Exception
BOM Tổng hợp được tạo bằng cách tham chiếu BOM Thành phần qua quan hệ BomListRelation.
UC-02 — Import / Export Excel BOM
Actor
NLG
Main Flow
• Export: GET /assign/bom-lists/export — xuất danh sách hiện hành theo filter.
• Import: POST /assign/bom-lists/import — parse Excel, validate, tạo BomListProduct hàng loạt.
Exception
Import sai format → trả 422 kèm mô tả lỗi.
UC-03 — Xoá BOM List
Actor
NLG (creator)
Precondition
BOM không đang được tham chiếu bởi YCBG / Báo giá đang hoạt động.
Main Flow
DELETE /assign/bom-lists/{id} → soft delete.
UC-04 — Gửi Yêu cầu báo giá (YCBG)
Actor
NV KD
Precondition
Dự án TKT tồn tại, đã chọn phiên bản giải pháp + hạng mục; tồn tại BOM Tổng hợp đã duyệt phù hợp.
Main Flow
1. Mở form YCBG trong dự án TKT. Gọi GET /assign/prospective-projects/{id}/pricing-requests/form-info → BE auto-pick BOM Tổng hợp status=4.
2. NV KD nhập nội dung → POST /assign/pricing-requests (status=1 Đang tạo).
3. Nhấn "Gửi" → POST /assign/pricing-requests/{id}/send → status 1 → 2 (Chờ XD giá).
4. Hệ thống thông báo tới mọi user có quyền "Xây dựng giá Bom giải pháp" trong phạm vi tổ chức.
Postcondition
PricingRequest status=2, danh sách YCBG của NLG cập nhật.
UC-05 — Tạo Báo giá từ YCBG
Actor
NLG
Precondition
PricingRequest status=2 (Chờ XD giá).
Main Flow
1. NLG mở màn "Chờ xây dựng giá" → chọn YCBG → POST /assign/quotations body {pricing_request_id}.
2. BE QuotationService::createFromRequest:
  — lockForUpdate PricingRequest (chống race).
  — Snapshot dữ liệu khách hàng (KH, MST, địa chỉ, email, người liên hệ, SĐT liên hệ) + thương mại (validity_days, warranty_time, payment_terms, currency).
  — Pre-populate QuotationProductPrice (giá=0, VAT%=0) cho từng BomListProduct.
  — Update PricingRequest status 2 → 3 (Đang XD giá).
  — Sinh mã BG-YYYY-NNNNN.
  — Log QuotationHistory::ACTION_CREATE.
Postcondition
Quotation status=1 (Đang tạo), đầy đủ dòng sản phẩm giá 0, khách hàng snapshot.
UC-06 — Nhập giá + VAT (save draft)
Actor
NLG (creator)
Precondition
Quotation status=1.
Main Flow
• PUT /assign/quotations/{id} → upsert estimated_price, quoted_price, vat_percent per dòng.
• Dòng CHA-có-CON: BE force quoted_price=0 (roll-up từ CON).
• Dòng CON: BE force vat_percent=0 (VAT chỉ áp CHA).
• POST /assign/quotations/{id}/apply-vat-bulk với body {vat_percent, mode ∈ {all, zero_only}}.
• Sau mỗi save, BE recomputeTotals cache total_vat_amount + total_after_vat.
• Log ACTION_SAVE_DRAFT hoặc ACTION_UPDATE_VAT_BULK.
Alt Flow
NLG có thể gọi POST /assign/quotations/{id}/calculate-level để xem trước cấp duyệt dự kiến dựa trên giá hiện tại.
UC-07 — Submit duyệt Báo giá
Actor
NLG (creator)
Precondition
Quotation status=1, tất cả dòng có quoted_price > 0.
Main Flow
1. POST /assign/quotations/{id}/submit.
2. BE recomputeTotals → gọi BomPriceApprovalConfigService::calculateApprovalLevel(totalSale_VND, profitMargin%) ra Level.
3. Level 1 → response {can_self_approve: true} — NLG tiếp tục gọi self-approve (UC-07.1).
4. Level 2/3 → status 1 → 2 (Chờ TP duyệt). Thông báo tới TP. Log ACTION_SUBMIT meta {level}.
Exception
Có dòng giá=0 → 422 "Còn dòng chưa có giá bán".
UC-07.1 — Self-approve (Level 1)
Actor
NLG
Precondition
UC-07 trả can_self_approve=true (Level 1), status=1.
Main Flow
POST /assign/quotations/{id}/self-approve → status 1 → 4 (Đã duyệt), set approved_by/approved_at; PricingRequest status 3 → 4 (Đã có báo giá). Thông báo NV KD + TP. Log ACTION_SELF_APPROVE.
UC-08 — TP duyệt
Actor
TP
Precondition
Quotation status=2.
Main Flow
POST /assign/quotations/{id}/tp-approve:
• Level 2 → status 2 → 4 (Đã duyệt), set approved_by/at, PricingRequest → 4. Log ACTION_TP_APPROVE. Thông báo NLG + NV KD.
• Level 3 → status 2 → 3 (Chờ BGĐ), set tp_approved_by/at. Log ACTION_TP_APPROVE_FORWARD. Thông báo BGĐ.
UC-09 — TP từ chối
Actor
TP
Main Flow
POST /assign/quotations/{id}/reject với body {reason} → status 2 → 1, clear approved_by/at, set rejected_reason. Log ACTION_REJECT meta {reason}. Thông báo NLG.
UC-10 — BGĐ duyệt
Actor
BGĐ
Precondition
Quotation status=3.
Main Flow
POST /assign/quotations/{id}/bgd-approve → status 3 → 4 (Đã duyệt), set approved_by/at; PricingRequest → 4. Log ACTION_BGD_APPROVE. Thông báo NLG + NV KD + TP.
UC-11 — BGĐ từ chối
Actor
BGĐ
Main Flow
POST /assign/quotations/{id}/reject từ status 3 → 1. Clear tp_approved_* + approved_*, set rejected_reason. Log ACTION_REJECT. Thông báo NLG + TP.
UC-12 — Cập nhật ghi chú kinh doanh
Actor
NV KD
Precondition
Quotation status=4 (Đã duyệt).
Main Flow
PUT /assign/quotations/{id}/sales-note → cập nhật sales_note mà không đổi status / giá.

6. Sơ đồ Swimlane — luồng end-to-end

Mỗi lane tương ứng 1 actor. Dòng chảy đi từ trên xuống dưới. Màu trạng thái: Đang tạo Chờ TP Chờ BGĐ Đã duyệt Từ chối

Ghi chú sơ đồ: mũi tên đứt nét đỏ = luồng từ chối / hoàn tác (quay lại status=1 cho NLG sửa). Mũi tên đứt nét đen = nhánh tự duyệt Level 1 bỏ qua 2 bước TP/BGĐ. Mỗi bước quan trọng đều sinh QuotationHistory + push notification.

11. Non-functional requirements

  • Performance: Cache total_sale / total_vat_amount / total_after_vat / profit_margin / approval_level trên quotations để list / filter / tab nhanh; tránh recompute khi query. Recompute đồng bộ khi save / submit / approve / VAT bulk.
  • Concurrency: createFromRequest và submit dùng transaction + lockForUpdate.
  • Auditability: Mỗi transition trạng thái sinh QuotationHistory với meta JSON đầy đủ.
  • Security: JWT (tymon/jwt-auth) + Laravel Sanctum; permission spatie; scope theo company_id/department_id/part_id.
  • Compatibility: PHP 7.4 + Laravel 8.65 + MySQL; Node 14.21.3 + Nuxt 2.14 + Vue 2 + Bootstrap-Vue 2.15.
  • Browsers: Chrome / Firefox / Safari / Edge phiên bản 2 năm gần đây.
